Copilot Studio Agent with Outlook Mailbox Connector

A quick walkthrough on getting your own Copilot Studio agent up and running with your Outlook mailbox. We'll cover creating the agent, plugging in the connector, building out the conversation topics, and shipping it — nothing too crazy, promise.

28 Apr 2026
Before You Start
  • A Microsoft 365 account with an Exchange Online mailbox (you probably already have this)
  • Copilot Studio license — comes bundled with Microsoft 365 Copilot, or grab a standalone one
  • Permissions to create agents in your environment (check with your admin if unsure)
  • WorkIQ already set up in your tenant — great news, this means semantic search is ready to go
1
Spin Up the Agent
~5 minutes
Step 1.1

Head over to copilotstudio.microsoft.com and sign in with your M365 account.

Step 1.2

Hit + Create on the left nav → pick "New agent".

Step 1.3

Give it a name — something like Mailbox Assistant works great. Add a quick description and set your language. Pro tip: you can tell it what to do in plain English right here, e.g., "Help me search, read, reply to, and forward my Outlook emails using natural language."

Step 1.4

Click Create and you're off. Copilot Studio will set up the agent with a default greeting and fallback topic — you'll customize these next.

2
Plug In the Outlook Connector
~5 minutes
Step 2.1

Inside your agent, head to Actions in the left nav → click "+ Add an action".

Step 2.2

Search for "Outlook" or "Office 365 Outlook" and select the Office 365 Outlook connector. Easy.

Step 2.3

You'll see a bunch of available actions. Add these ones — they're the core toolkit you'll need:

Get emails (V3) Get email (V2) Send an email (V2) Reply to email (V3) Forward email (V2) Flag email (V2)
Step 2.4

When it asks you to sign in, use the M365 account whose mailbox you want the agent to tap into. This kicks off the OAuth flow and sets up the connection. The connector will request Mail.Read, Mail.Send, and Mail.ReadWrite — standard stuff.

Bonus — WorkIQ Integration

Since WorkIQ is already good to go in your tenant, you can add it as a knowledge source under the agent's Knowledge tab. This gives the agent superpowers — semantic search across all your M365 stuff (emails, files, Teams chats) beyond what the Outlook connector alone can do.

3
Build the Conversation Topics
~20 minutes
Step 3.1

Go to Topics in the left nav and click "+ Add a topic" → "From blank". You'll want one topic per use case — check out the Recommended Topics below for the full list.

Step 3.2 — Topic Structure

Every topic follows the same basic flow:

1. Trigger phrases — the things a user might say to kick it off (add 5–10 variations)
2. Ask a question — grab any info you need (sender name, keyword, date range)
3. Call an action — fire off the Outlook connector action (e.g., Get emails)
4. Show a message — display results back in a card or text block
5. Condition branches — handle "nothing found" or "multiple matches" gracefully

Step 3.3 — Variables & Slots

Use topic variables to capture what the user says. For the "Search Emails" topic, you'd create variables like SearchKeyword, SenderName, and DateRange. Then map those to the Outlook connector's $filter and $search params in the action node. Copilot Studio makes this pretty visual — just drag and connect.

Shortcut — Generative Orchestration

In Settings → Generative AI, flip on "Generative orchestration". This is the magic sauce — it lets the AI figure out which topic and action to use based on what the user says. Way less time spent writing trigger phrases, and it handles edge cases you'd never think of.

Topics Worth Building
Email Triage
Fetch unread/flagged emails and present a categorized summary. Use the Get emails action with $filter=isRead eq false. Display results as a numbered list with sender, subject, and received time.
"Check my inbox" "What's new in my email" "Show unread emails"
Search Emails
Search by sender, subject keyword, or date range. Use Get emails with $search KQL parameter. Ask the user for the search term, then display matching results.
"Find emails from Sarah" "Emails about budget" "Messages from last week"
Read Email Details
After search/triage, let the user select an email by number. Use Get email (V2) with the message ID to fetch the full body. Display the complete content in a formatted card.
"Read email #3" "Open that message" "Show me the details"
Reply to Email
Draft a reply using AI-generated content based on context. Ask the user for the reply intent, generate a draft, show it for confirmation, then call Reply to email (V3).
"Reply to that email" "Respond saying I agree" "Send a reply"
Forward Email
Forward a selected email to a specified recipient. Ask for the recipient name/email, optionally add a comment, then call Forward email (V2).
"Forward this to my manager" "Send this to the team" "Forward email #2"
Compose New Email
Collect recipient, subject, and body from the user. Optionally let AI draft the body from a brief description. Call Send an email (V2) after user confirms the draft.
"Send an email to John" "Draft a message" "Compose a new email"
Outlook Connector Actions Reference
Action What It Does Key Parameters
Get emails (V3) Retrieve a list of emails matching filter/search criteria $filter, $search, $top, $orderby, folderPath
Get email (V2) Fetch full content of a single email by message ID messageId, includeAttachments
Send an email (V2) Compose and send a new email to, subject, body, cc, bcc, importance
Reply to email (V3) Send a reply to an existing email thread messageId, comment, replyAll
Forward email (V2) Forward an email to specified recipients messageId, toRecipients, comment
Flag email (V2) Set flag status on an email (flagged/complete/clear) messageId, flagStatus
Move email (V2) Move email to a different folder messageId, destinationFolder
Mark as read/unread Toggle read status on an email messageId, isRead
4
Take It for a Spin
~10 minutes
Step 4.1

Hit the "Test your agent" button (bottom-left). This pops open a chat panel where you can talk to your agent live.

Step 4.2

Try some real prompts and see how it handles them:

"Show me my unread emails"
"Find emails from Sarah about the budget report"
"Reply to email #1 saying I'll review it by Friday"
"Forward that email to alex@contoso.com"
"Send a new email to the project team about the milestone update"

Step 4.3

Check that the agent picks the right Outlook action, shows useful results, and doesn't trip up on edge cases. Tweak your trigger phrases and prompts based on what you see — it's an iterative thing.

5
Ship It
~5 minutes
Step 5.1

Happy with how it works? Hit Publish (top-right). Your agent is now live and ready to roll.

Step 5.2 — Channel Options

Head to Settings → Channels to pick where you want to deploy it:

Microsoft Teams Web Chat (embed) Copilot for M365 SharePoint
Step 5.3

For Teams (probably the best option): click the Teams channel → follow the quick setup to install it as a Teams app. Your users can then just chat with it directly inside Teams. Done!

6
Quick Security Sanity Check
Ongoing
Step 6.1

Auth model: The Outlook connector uses delegated permissions — meaning the agent acts as the signed-in user, not as some service account. No admin consent needed unless your tenant specifically requires it. Each person authenticates the first time they use the agent.

Step 6.2

Your data stays put: All email data stays within the M365 tenant boundary. The agent doesn't store email content anywhere — it reads and acts in real-time through the connector. Nothing leaves the compliance perimeter.

Step 6.3

DLP check: If your tenant has Data Loss Prevention policies, just make sure the Outlook connector isn't blocked. Quick check: Power Platform Admin Center → Data policies.

Good to Know

Because it uses delegated auth, the agent can only see what the user themselves can already see. It can't snoop into other people's mailboxes. That's the safest model for personal mailbox use cases — no extra risk.
